# AetoxOS: Multi-Agent Evolution Blueprint

พิมพ์เขียวเชิงสถาปัตยกรรมสำหรับการขยายระบบไปสู่ Multi-Agent ที่สามารถทำงานต่อเนื่องและจำบริบทได้แม่นยำ

---

## 1. องค์ประกอบของระบบ (The Trinity)

### A. The Master Manager (Discord Bot)
- **หน้าที่:** รับคำสั่งจากคุณกิต และใช้ AI ตัดสินใจว่างานนี้เป็น "งานเดี่ยว" (Direct Step) หรือ "งานใหญ่" (Multi-Agent Task)
- **การส่งต่อ:** หากเป็นงานใหญ่ จะทำการสร้าง `TaskID` และส่งต่อให้ Orchestrator ทันที

### B. The Orchestrator (ผู้จัดการวง)
- **หน้าที่:** ควบคุม Flow การทำงานระหว่าง Agent ตัวอื่นๆ
- **State Management:** ถือครอง `Global_Task_State` เพื่อดูว่าตอนนี้ใครกำลังทำงาน และงานไปถึงไหนแล้ว

### C. Specialized Agents (เหล่าผู้เชี่ยวชาญ)
- **Planner:** วางแผนงานย่อย (To-Do List)
- **Executor:** ลงมือใช้ Tools (เหมือนบอทที่เรามีตอนนี้)
- **Researcher:** ค้นหาข้อมูลเชิงลึก (ทั้งในเครื่องและในเว็บ)
- **Critic:** ตรวจสอบความถูกต้องก่อนส่งมอบงาน

---

## 2. ระบบความจำบริบท (Context Persistence Model)

เพื่อให้บอทไม่ "หลงทาง" เราจะใช้ระบบ **"Shared Working Memory"** ดังนี้:

| ส่วนประกอบ | หน้าที่ในระบบ Multi-Agent |
| :--- | :--- |
| **Task State** | เก็บสถานะปัจจุบัน (เช่น Step 2 จาก 5) และเป้าหมายสูงสุด |
| **Memory Buffer** | เก็บข้อมูลที่ Agent แต่ละตัวหามาได้ เพื่อให้ตัวถัดไปนำไปใช้ต่อได้ |
| **Agent Logs** | บันทึกว่าใครทำอะไรไปบ้าง เพื่อให้ระบบ Critic สามารถย้อนกลับไปตรวจสอบได้ |

---

## 3. Workflow การทำงาน (Sequence)

1. **User:** "สรุปโปรเจกต์ A, จัดไฟล์ที่เกี่ยวข้อง, และหาข้อมูลคู่แข่งในเว็บมาเปรียบเทียบที"
2. **Master:** ตัดสินใจส่งให้ Orchestrator
3. **Planner:** ร่างแผนงาน 3 ขั้นตอน
4. **Executor:** ไปหาไฟล์โปรเจกต์ A และจัดระเบียบ (จำพาธไว้ใน Shared Memory)
5. **Researcher:** เอาชื่อโปรเจกต์จาก Shared Memory ไปค้นหาในเว็บ
6. **Critic:** รวมผลลัพธ์มาเช็คว่าครบถ้วนไหม
7. **Master:** ส่งสรุปสุดท้ายให้คุณกิตใน Discord

---

## 4. มาตรฐานที่ต้องมีก่อนไปถึงจุดนั้น
- **Clean Tools:** เครื่องมือต้องเป็นมาตรฐาน Gold Standard (ที่เราเพิ่งทำเสร็จ) เพื่อให้ Agent ทุกตัวเรียกใช้ได้อย่างแม่นยำ
- **Fast Reasoning:** ใช้โมเดลที่คิดเร็วและแม่นยำ (เช่น Qwen2.5:14b หรือโมเดลที่ใหญ่กว่า)
- **Robust Error Handling:** ถ้า Agent ตัวใดตัวหนึ่งพลาด ระบบ Orchestrator ต้องสามารถสั่ง Rollback หรือแก้ไขแผนได้อัตโนมัติ
